# StudyOS Docs

This folder contains focused reference documents for planning and implementing StudyOS.

Use `plan.md` as the full roadmap. Use these docs when working on a specific part of the product.

## Documents

- `product-reference.md` - product thesis, principles, user journey, and MVP success criteria.
- `mvp-scope.md` - v0.1 must-have, should-have, later, and non-goal lists.
- `technical-architecture.md` - suggested stack, app structure, and integration boundaries.
- `data-model.md` - MVP tables, relationships, and important fields.
- `ai-pipeline.md` - file processing, AI generation, failure states, and reliability rules.
- `planner-rules.md` - deterministic planner behavior, inputs, outputs, and adaptation rules.
- `development-roadmap.md` - phased implementation plan with verification checks.
- `open-decisions.md` - unresolved product and technical decisions.

## Current Product Focus

StudyOS v0.1 should prove the core study loop:

1. Upload study material.
2. Extract useful study metadata.
3. Add deadlines and availability.
4. Generate a realistic study plan.
5. Complete or miss sessions.
6. Adapt the plan from progress.

Avoid expanding into a full task app, calendar app, note app, or AI tutor before this loop works reliably.
